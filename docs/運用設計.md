# 進捗をいつ記録するか

進捗は細かく記録できるほど理想ですが、入力回数が増えるほど現場の負担が増えます。

そこで TaskBoard では
「必要な情報は押さえつつ、入力回数はできるだけ減らす」
落としどころを探しました。

-   却下案: 1 ケースごとに記録
    細かく把握できますが、私なりの肌感では、手を止める回数が多くなり過ぎて、作業の流れを邪魔しやすいと考え、不採用にしました。

-   採用案: 1 本単位で記録
    「1 本分がまとまって冷蔵庫に搬入されたタイミング」で、
    まとめて袋数の進捗を記録する想定です。

    作業の区切りとして自然で、ケース単位より入力頻度を抑えられます。

また、搬入と搬出の両方で本数を記録すれば、
「入れたはずの本数」と「実際の本数」の食い違いにも気づきやすく、ボトルネックの把握にも役立つ可能性があると考えました。

結論として、記録単位は「1 ケース」ではなく「1 本」が、現場負担と情報量のバランスが良いと判断しました。

# 進捗をどこで記録するか

次に悩んだのは「どこで入力するのが一番自然か」です。
前段で、進捗記録のタイミングを「1 本を冷蔵庫に搬入/搬出するとき」としたため、
入力場所は冷蔵庫の出入口付近が最も自然だと考えました。

冷蔵庫の出入口は複数の班が必ず通る場所です。
ここに入力端末を置けば、作業動線から大きく外れず、少ない台数でも記録できると考えました。

# 進捗をどうやって記録するか

進捗入力の方式として、スキャン機能つきスマホ運用を想定し、次の 2 点を比較しました。

-   入力方式: 手入力 / スキャン
-   端末構成: 共用 1 台 / 班ごとに複数台

前提として、現場での目視確認や声掛けは、システム導入後も一定残る想定で整理しました。

# 進捗をどうやって記録するか: 手入力 vs スキャン

前提として、冷蔵庫出入口付近にスキャン機能付きのスマホを置き、
「どの班が・何を・何袋を・搬入/搬出したか」を記録する運用を想定しました。

また、ラベルのバーコードは「商品種別の共通コード」程度にとどめ、「1 本の中での順番」までは区別しない想定です。

-   スキャン入力の利点
    操作ステップが少なく、速く、入力ミスも減らしやすい。

-   スキャン入力の懸念
    ラベルの紛失、汚れ、折れなどで読み取れない場合に入力が止まる。

-   手入力の利点
    ラベル状態に依存せず入力できる。

-   手入力の懸念
    操作ステップが増え、遅くなり、入力ミスも増えやすい。

また、現場で「置き場所」が問題になる場面もあると感じました。

経験上、ライン班のリーダーが
「ある 1 本をある場所に運んだ人は誰か」
と強い口調で確認していたことがあり、名乗り出なかったため監視カメラで追う話になったことがあります。

ここから、運搬先には複数の候補があり、誤った場所に置くと都合が悪くなるため、
「どこに置いたか」を強く重視しているのだと気づきました。

そのため、スキャンにより「誰が・いつ・どこへ搬入 / 搬出したか」を記録できれば、
置き場所ミスの抑止や、発生時の追跡がしやすくなり、再発防止にも寄与すると考えています。

結論として、通常はスキャン入力を基本とし、ラベル不良や想定外のケースは手入力に切り替える「ハイブリッド運用」を想定しました。

# 進捗をどうやって記録するか: 冷蔵庫に 1 台 vs 班ごとスマホ

冷蔵庫には複数の班が出入りし、それぞれが自分の搬入・搬出を記録する前提で比較しました。

-   共用 1 台（冷蔵庫に設置）
    初期費用や充電、故障対応のコストは小さい一方で、入力のたびに班の画面切り替えが必要です。
    切り替え忘れによる誤登録や、混雑時の入力待ちが発生する可能性があります。

-   班ごとに専用スマホ
    班の画面を出しっぱなしにでき、切り替え不要で誤登録リスクが減ります。
    一方で端末台数が増え、初期費用、紛失、充電管理のコストが上がります。

結論として、初期は共用 1 台で開始し、必要性と費用対効果が見えた段階で班ごと設置へ拡張する想定です。
