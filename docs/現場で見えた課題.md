# 人員配置

タイミーとして班を移動した際、社員の方の指示で別の班へ向かったものの、
「今はいらない」と言われ、行き先が何度か変わった経験があります。

このとき、
「工場全体の Task の進捗や、どこが詰まりそうか」をその場で共有・把握するのは、
意外と難しい場面があるのではないかと感じました。

もしそうであれば、全体状況を整理して見える化することで、
人員配置の判断を支えられる余地があると考えました。

# 製品の準備が間に合わない

一言で言えば、社員の方が「出荷時間に向けた調整」を強く意識しており、
その負担が大きそうだという印象を持ちました。

例えば、
「スーパー向けのトラックが到着したとき、製品の準備が間に合わず、
発送班の方が対応に追われることがある」と伺ったことがあります。

また現場でも、
班リーダーの方々と集まり Task の優先順位を相談している場面や、
出荷時間が迫る中でモニター越しに現場を見ながら
「このままだと 19 時に間に合わない」と状況を共有している場面を見聞きしました。

もちろん私の立場では、
準備が間に合わない原因が社内システムにあるかどうかを断定できません。
ただ、もしシステムが一因になっているなら、
Task の優先順位や人員配置の判断材料が「5 分でも早く揃う」ことに価値があると感じました。

そこで TaskBoard では将来的に、
トラックの到着予定時間や班ごとの Task の進捗をインプットとして扱い、
シンプルなロジックで「調整の判断材料」を提示することを狙っています（現状は構想段階です）。
